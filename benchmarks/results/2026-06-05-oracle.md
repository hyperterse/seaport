# Seaport vs Harbor Benchmark: Sandboxed Oracle Task

Date: 2026-06-05

Environment:

- OS: macOS arm64
- Docker: 29.4.0 via OrbStack
- Harbor: 0.13.1
- Seaport: local release build from this repository

Task: `benchmarks/tasks/basic-oracle`

Iterations: `3`

| Tool | Backend | Successful runs | Mean seconds |
| --- | --- | ---: | ---: |
| Seaport | Docker | 3 | 1.670776 |
| Harbor | Docker-backed oracle execution | 3 | 15.993786 |

Seaport was faster in this sandboxed run.

Speedup ratio: `9.573x`

## Commands

```sh
target/release/seaport run -p benchmarks/tasks/basic-oracle -a oracle --backend docker --jobs-dir <tempdir>/jobs
```


## Caveat

This benchmark compares oracle execution on the same task. Seaport uses its
hardened Docker backend with no network, dropped capabilities, a read-only root
filesystem, non-root execution, and CPU, memory, swap, PID, and wall-clock
limits. Harbor uses its Docker-backed oracle task execution. The result is a
valid measurement for this benchmark path, but it is not yet a claim that
Seaport has feature parity with Harbor's full registry, viewer, or agent
integration surface.
